自动编译PB文件，并添加到项目

Philip.Lee

2017-07-15

iOS

项目中使用了Google ProtoBuf, 每次后台新增PB文件的时候，前端都需要编译新增的PB，转化为OC文件，如果已经在项目中存在了，则需要去查找替换的已有的Objective-C文件。文件少可能直接搜索就可以了。但是新增的PB文件多了，项目大了，就比较难以管理，而且项目我们使用的ARC模式，但是PB文件需要MRC, 所以每次新增PB文件，不止要手动添加到项目中，而且需要手动给每个编译好的OC文件添加编译参数为-fno-objc-arc，确实有点麻烦。鉴于Pod可以管理项目文件，自动添加文件到项目中。所以研究了下Pod的更新项目的基本原理。编写了基于Ruby的脚本（Pod更新项目的脚本是基于Ruby的）。

准备工具

  • protoc： 转换PB文件为OC文件的编译器（ 具体使用方法看这里, 使用要看版本，各个版本编译之后的OC文件的有些不一样，前端和后台的protoc编译器的版本需要一致 ）
  • xcodeproj : 更新Xcode项目的脚本框架，需要Ruby环境 （ 具体看这里, 安装命令： sudo gem install xcodeproj ）
  • SVN: 该脚本是基于PB文件的SVN库的更新来做相应处理的

有待改进

因为当前项目中的PB文件，很多项目都在共用一份，也就是所有项目都是基于这一份PB文件开发，所以这个PB仓库会越来越大，所以难免会添加本项目不必要的文件，可以有以下解决方案：

  • a) 把各个项目中的PB文件独立出来，自成一个独立仓库。 这种多个项目共有的PB文件就会有个实例了。

  • b) 在脚本中添加文件夹黑名单。添加文件黑名单有点不现实，而且文件也是逐渐在增加的。

  • c) 把各个项目中的PB文件使用独立文件夹，共用的为独立共享文件夹。但是这种不便于管理。
