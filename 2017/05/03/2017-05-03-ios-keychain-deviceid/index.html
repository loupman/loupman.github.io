iOS10.3使用keychain存取deviceID

Philip.Lee

2017-05-03

iOS

背景

鉴于iOS10.3的beta，当APP被卸载之后，如果没有系统会删除APP keychain存储的数据. 如果使用keychain group时，只有当这个group中所有的APP被卸载之后，才会删除这个group的所有keychain 数据（已测试）
但是iOS10.3 正式版本，并没有包括这个新特性（在10.3.2上已测试）

在iOS10之后，系统新增了一个公共group kSecAttrAccessGroupToken，可以使用这个group进行存储必要的数据，但是这个存在一定的安全问题，所有的APP都有访问这个公共group的权限，都可以对数据进行增删改。

需求

项目中有需求，需要跟踪用户，给用户的设备设置一定的权限，所以需要标识用户设备。

原理-使用Keychain

对于keychain的操作，APPLE提供了专用的标准API来增删改查。 只需要提供查询条件（表，主键，是否有group）

  1. 添加一个项数据， attributes 是一个NSDictionary查询条件

  2. 更新一项数据，query：表示查询条件，attributesToUpdate：更新的值。当通过query查询到数据后，就可以使用attributesToUpdate更新数据了。

  3. 删除一项数据

  4. 搜索

查询的NSDictionary主要包括，查询的表和主键。keychain在iOS持久化使用的Sqlite存储。
在决定去哪个标查询数据，是以字段kSecClass,决定，字段kSecClass有以下几种取值，对应了需要把数据存储在哪个表里。

kSecClassGenericPassword 对应的表：genp， 一般密码
kSecClassInternetPassword 对应的表：inet， 网络密码
kSecClassCertificate 对应的表：cert， 证书
CFTypeRef kSecClassKey 对应的表：keys， 密钥
CFTypeRef kSecClassIdentity 对应的表：没找到 身份证书(带私钥的证书)。

每个表拥有的字段都是以 kSecAttr开头定义的， 都是系统自己维护的，这些字段都指定了数据类型，所以什么字段放什么类型的值都是固定的，没法更改。

*值得注意的是，最常用的kSecClassGenericPassword表 和 kSecClassInternetPassword表 的主键是kSecAttrAccount和 kSecAttrService，所以新保存数据的时，这两项在已有项中不能重复，否则保存失败。 *

用户保存自定义数据常会使用字段 kSecValueData（代码中我使用的是这个）, kSecValueRef, kSecValuePersistentRef。 这三个常量都对应表的data字段，但在从数据库取出后，会转为不同的数据类型。 这个字段是会加密保存的。

实现想法

鉴于我们APP只是标识用户手机，并没有涉及用户的一些隐私数据，别人拿到也是并没有什么大的作用，那么我们可以生成一个id 存储在这两个地方，APP独有的keychain，公共group的keychain中。获取的时候，依次从这两个地方取值，如果都没有取到，那么则生成一个新的id。

逻辑如下：

获取逻辑：

  1. 先从APP独有keychain中获取
  2. 如果没有，则看是否系统版本是否是10及以上
  3. 如果是，则从公共group中获取deviceID
  4. 都没有，则生成一个deviceID，保存device ID 到这两个地方。

生成存储逻辑：

  1. 如果APP可以使用广告IDFA，则使用IDFA进行跟踪
  2. 用户未开启，或者APP未接入广告，则使用系统生成UUID
  3. 保存DeviceID 到keychain中
